DOM、SAX、JDOM、DOM4J

1、DOM把整个XML文件当作一个整体对象。放到内存中去。

2、SAX把一个XML文档当作一个流媒体来进行处理（防止XML文件过大），读一点解析一点，专门用来处理大型的XML文件。
（缺点：不能对文档进行随机访问
SAX是只读的
文档只能遍历一次。）

3、JDOM包含了以上两种解析方式

4、DOM4J也包含了DOM和SAX两种解析方式（同JDOM的区别：DOM4J的性能是最好的）

解析XML的子节点、根节点：
根节点就是文档开始的第一个标签，例如：Hibernate-mapping

获取根节点实例：public static void main(String[] args) throws Exception{
SAXReader reader = new SAXReader();
Docunment document = reader.read()
}

xpath:在XML文件中找到指特定节点的路径的语法。

DOM4J生成、修改XML文件。
